## Directory Purposes

**src/agent/:**
- Purpose: Core AI agent (Claude SDK wrapper), multi-agent orchestration, planning, self-improvement
- Contains: PersonalAssistant class, orchestrator for task decomposition, team system (AgentManager, TeamRouter, TeamBus), prompt caching, security hooks
- Key files:
  - `assistant.ts` - PersonalAssistant, session management, tool execution
  - `orchestrator.ts` - Task decomposition, parallel execution, result synthesis
  - `team-router.ts` - Inter-agent message routing with dependency graph
  - `team-bus.ts` - Async message queue for team communication
  - `agent-manager.ts` - Load and manage agent YAML configs
  - `hooks.ts` - Security enforcement (tool permissions, approval workflows)
  - `prompt-cache.ts` - Reuse expensive contexts via SDK caching
  - `tool-loop-detector.ts` - Detect infinite tool recursion
  - `self-improve.ts` - Autonomous improvement cycles
  - `daily-planner.ts` - Goal/task planning agent
  - `workflows-runner.ts` - Execute workflow files from vault
  - `link-extractor.ts` - Extract wikilinks from responses
  - `profiles.ts` - Load role profiles (function specifications for agents)
  - `role-scaffolds.ts` - Generate prompts from role definitions

**src/channels/:**
- Purpose: Multi-platform message handling (Discord DMs, Slack, Telegram, WhatsApp, webhooks)
- Contains: Protocol-specific adapters, streaming handlers, command parsers, bot managers
- Key files:
  - `discord.ts` - Discord.js bot (DM-only, slash commands, streaming)
  - `discord-bot-manager.ts` - Manages Discord bot lifecycle
  - `discord-utils.ts` - Message chunking, embed formatting
  - `slack.ts` - Slack bolt adapter
  - `slack-bot-manager.ts` - Manages Slack app lifecycle
  - `slack-utils.ts` - Slack formatting helpers
  - `telegram.ts` - Telegram grammy adapter
  - `whatsapp.ts` - WhatsApp Twilio integration
  - `webhook.ts` - Generic webhook receiver for external events
  - `discord-agent-bot.ts` - Team-aware Discord agent bot
  - `slack-agent-bot.ts` - Team-aware Slack agent bot

**src/cli/:**
- Purpose: Command-line interface, daemon control, dashboard server, interactive chat
- Contains: Commander.js commands, Express dashboard SPA, setup wizard, tunnel manager
- Key files:
  - `index.ts` - CLI entry point, command definitions (start, stop, restart, status, doctor, config)
  - `dashboard.ts` - Express SPA server (localhost:3030), JSON APIs for metrics/cron/memory
  - `chat.ts` - Interactive terminal chat interface
  - `cron.ts` - Cron job management commands
  - `setup.ts` - Initial configuration wizard
  - `tunnel.ts` - TunnelManager for remote access via Cloudflare/Ngrok

**src/gateway/:**
- Purpose: Message routing, session management, concurrency control, error classification
- Contains: Core router, schedulers, lane controller, notification dispatcher
- Key files:
  - `router.ts` - Gateway class: session lookup, chat routing, error classification
  - `heartbeat-scheduler.ts` - Periodic check-ins (configurable interval and active hours)
  - `cron-scheduler.ts` - Parses CRON.md and executes scheduled jobs
  - `notifications.ts` - NotificationDispatcher: sends to all active channels
  - `lanes.ts` - Lane controller (concurrency pooling): chat, cron, heartbeat, self-improve, team

**src/memory/:**
- Purpose: Vault indexing, semantic search, conversation memory, graph relationships
- Contains: SQLite FTS5 database, chunking logic, MMR reranking, graph store
- Key files:
  - `store.ts` - MemoryStore: FTS5 database, schema, sync vault to index
  - `chunker.ts` - Parse markdown, extract frontmatter, split into searchable chunks
  - `search.ts` - Query FTS5, rerank with MMR, format for agent context
  - `mmr.ts` - Maximal Marginal Relevance reranking (diversity)
  - `graph-store.ts` - Wikilink graph and connection tracking

**src/security/:**
- Purpose: Tool permission enforcement, malicious pattern detection
- Contains: Permission framework, pattern scanner, integrity checks
- Key files:
  - `scanner.ts` - Detect malicious code patterns, SQL injection, command injection
  - `patterns.ts` - Regex patterns for vulnerability scanning
  - `integrity.ts` - File integrity checking

**src/tools/:**
- Purpose: Model Context Protocol server exposing 27+ tools to Claude SDK subprocess
- Contains: MCP server, tool handlers, vault/memory access, external API clients
- Key files:
  - `mcp-server.ts` - Standalone stdio server; defines tools (memory, vault, Outlook, Salesforce, GitHub, RSS, etc.)

**src/vault-migrations/:**
- Purpose: Schema evolution and vault structure migrations
- Contains: Migration runners, helpers, type definitions
- Key files:
  - `runner.ts` - Migration execution engine
  - `types.ts` - Migration interface definitions
  - `helpers.ts` - Shared migration utilities

## Naming Conventions

**Files:**
- `camelCase.ts` for modules and source files
- `UPPERCASE.md` for vault markdown (SOUL.md, MEMORY.md, CRON.md, etc.)
- Descriptive names: `orchestrator.ts` not `planner.ts`, `heartbeat-scheduler.ts` not `pulse.ts`
- Hyphenated for compound names: `tool-loop-detector.ts`, `discord-bot-manager.ts`

**Directories:**
- Lowercase hyphenated: `src/vault-migrations/`, `src/gateway/`
- Numeric prefixes for vault structure: `01-Daily-Notes/`, `02-People/` (sorting order)
- System directory always `00-System/`

**Exports:**
- Named exports for classes, interfaces, utilities
- Default exports rarely used (prefer named)
- Index files (`index.ts`) in some modules export the primary class

**Variables & Functions:**
- camelCase for variables, functions, parameters
- UPPER_SNAKE_CASE for constants (e.g., `CHAT_TIMEOUT_MS`, `MAX_CONCURRENT_STEPS`)
- PascalCase for class names and type names

**Types:**
- PascalCase for interfaces, types, classes
- Suffix `Callback` for function types (e.g., `OnTextCallback`)
- Suffix `State` for state interfaces (e.g., `SessionState`)
- Suffix `Config` for configuration interfaces

## Where to Add New Code

**New Feature (Agent Capability):**
- Primary code: `src/agent/` (new file or extend `assistant.ts`)
- Hooks/Security: Add to `src/agent/hooks.ts` if permission enforcement needed
- Tests: `tests/agent/` following same filename
- Example: Adding a new workflow → `src/agent/my-workflow.ts` exported from main assistant

**New Channel (Platform Integration):**
- Implementation: `src/channels/my-platform.ts`
- Manager: `src/channels/my-platform-bot-manager.ts`
- Utils: `src/channels/my-platform-utils.ts`
- Registration: Add to `src/index.ts` daemon initialization
- Tests: `tests/channels/my-platform.test.ts`

**New Tool (MCP Tool Exposed to Claude):**
- Handler: Add function to `src/tools/mcp-server.ts`
- Define schema (input/output via Zod)
- Register in `tools` array passed to McpServer.setRequestHandler()
- Test via `npm run mcp` (spawns server standalone)

**CLI Command:**
- Implementation: `src/cli/my-command.ts`
- Export function: `export async function cmdMyCommand(options: ...): Promise<void>`
- Register: Add to commander in `src/cli/index.ts` via `.addCommand()`

**Utilities / Helpers:**
- Shared memory functions: `src/memory/search.ts`
- Shared security: `src/security/scanner.ts`
- Shared vault operations: Add to `src/agent/assistant.ts` or create `src/vault-utils.ts`

**Types / Interfaces:**
- Shared types: `src/types.ts`
- Module-local types: Define at top of module file

## Special Directories

**vault/:**
- Purpose: Example vault structure showing valid markdown layout
- Generated: No (user-created and maintained)
- Committed: Yes (contains templates and examples)
- Real data lives in `~/.clementine/vault/`

**dist/:**
- Purpose: Compiled JavaScript output from TypeScript
- Generated: Yes (build via `npm run build`)
- Committed: No (in .gitignore)
- Entry point for running: `node dist/cli/index.js`

**.planning/:**
- Purpose: GSD (Get Stuff Done) planning documents
- Generated: Yes (by `/gsd:map-codebase`)
- Committed: Yes (guides future work)
- Contents: ARCHITECTURE.md, STRUCTURE.md, STACK.md, CONVENTIONS.md, TESTING.md, CONCERNS.md, INTEGRATIONS.md

**node_modules/:**
- Purpose: Installed dependencies
- Generated: Yes (via `npm install`)
- Committed: No (lockfile committed instead)
